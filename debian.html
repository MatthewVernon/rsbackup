rsbackup and Debian

In the .deb package, the default /etc/rsbackup/config is as follows:

# Location of lockfile
lock /var/run/rsbackup.lock

# User configuration
include /etc/rsbackup/local

# Hosts
include /etc/rsbackup/hosts.d

Normally you would not edit this file. Instead, put local configuration in /etc/rsbackup/local and a file for each host to back up into /etc/rsbackup/hosts.d.

There is also a configuration file for the cron jobs, /etc/rsbackup/defaults. The default is as follows:

#
# Set daily=true or daily=false to enable or suppress a daily backup run.
#
daily=true

#
# Set email=ADDRESS to email a report from the daily backup run to
# ADDRESS.
#
email=root

#
# Set hourly="HOST1 HOST2..." to back up named hosts hourly instead of daily.
#
hourly=""

#
# Set prune="OPTIONS" to control weekly pruning options.
#
prune="--prune --prune-incomplete"

Leave daily set to true if youâ€™re happy with automatic backups every night or set it to false if you have some other arrangement for initiating backups. See also /etc/cron.daily/rsbackup.

Set email to the destination address for reports from the daily backup run, or comment out the line entirely to suppress email reports. See also /etc/cron.daily/rsbackup.

Set hourly to a list of hosts to attempt to back up hourly. This is intended to be used to opportunistically back up hosts that are often down, for instance laptops and personal desktops. See also /etc/cron.hourly/rsbackup.

Modify prune to tune the weekly pruning options. See also /etc/cron.weekly/rsbackup.
